# ChatSensei Lite - 演示指南

本指南帮助你在课程演示中展示 ChatSensei Lite 的所有核心功能。

---

## 🎬 演示流程（建议 5-10 分钟）

### 1️⃣ 介绍项目背景 (1分钟)

**说明内容：**
- ChatSensei Lite 是一个智能消息助手原型
- 帮助用户快速生成不同风格的聊天回复
- 使用简单的强化学习来学习用户偏好
- 支持在线（OpenAI API）和离线（启发式）两种模式

---

### 2️⃣ 启动应用 (30秒)

**操作步骤：**
```bash
cd chatsensei_lite
./start.sh          # macOS/Linux
# 或 start.bat     # Windows
```

**展示要点：**
- ✅ 一键启动，自动检查环境
- ✅ 显示配置信息（OpenAI API 状态）
- ✅ 服务器成功启动

### 4️⃣ 功能演示 - 多风格建议生成 (2分钟)

**使用测试案例：**
```
朋友：周末一起去爬山吧！天气很好哦！
```

**展示三种风格：**

🎩 **礼貌风格**
- 特点：正式、尊重、得体
- 示例：感谢你的邀请！[基于语气的回复]

😄 **幽默风格**
- 特点：轻松、有趣、带emoji
- 示例：哈哈，[基于语气的回复] 😂

💬 **直接风格**
- 特点：简洁、清晰、直截了当
- 示例：[简短的回复]

**强调要点：**
- 三种风格满足不同社交场景需求
- 启发式模式基于语气生成合理回复
- 如果配置了 OpenAI API，会生成更自然的建议

---

### 5️⃣ 功能演示 - 偏好学习 (3分钟)

**演示强化学习机制：**

**初始状态：**
```
礼貌: 1.0  |  幽默: 1.0  |  直接: 1.0
```

**操作 1: 选择礼貌风格**
- 点击礼貌建议的 "使用 👍"
```
礼貌: 2.0  |  幽默: 1.0  |  直接: 1.0
```

**操作 2: 再次选择礼貌风格**
- 点击礼貌建议的 "使用 👍"
```
礼貌: 3.0  |  幽默: 1.0  |  直接: 1.0
```

**操作 3: 不喜欢幽默风格**
- 点击幽默建议的 "不好 👎"
```
礼貌: 3.0  |  幽默: 1.0  |  直接: 1.0
```
（最小值保护，不会降到 1.0 以下）

**强调要点：**
- 使用**上下文赌博机**（Contextual Bandit）算法
- 正面反馈：权重 +1.0
- 负面反馈：权重 -0.5（最小保持 1.0）
- 实时更新，立即生效
- 适合个性化推荐场景

---

### 6️⃣ 技术架构讲解 (2分钟)

**后端技术栈：**
- 🐍 Python + FastAPI（异步高性能）
- 📝 Pydantic（数据验证）
- 🎨 Jinja2（模板引擎）
- 🤖 OpenAI API（可选）

**前端技术栈：**
- 📄 HTML5 + Vanilla JavaScript
- 🎨 Pico.css（轻量级 CSS 框架）
- ⚡ Fetch API（异步请求）

**核心算法：**
- **语气检测**：基于关键词匹配
- **建议生成**：双模式（LLM / 启发式）
- **偏好学习**：简单的上下文赌博机

## 💡 可能的提问及回答

### Q1: 为什么选择上下文赌博机而不是更复杂的 RL？
**A:** 
- 上下文赌博机适合推荐系统
- 实现简单，易于理解
- 计算效率高，适合实时应用
- 对于原型项目来说足够有效

### Q2: 如果没有 OpenAI API 怎么办？
**A:** 
- 系统自动回退到启发式模式
- 基于语气检测生成规则化建议
- 保证系统始终可用
- 展示了健壮的系统设计

### Q3: 偏好数据为什么不持久化？
**A:** 
- 当前使用内存存储（原型阶段）
- 易于实现和演示
- 未来可轻松迁移到数据库
- 重启即可重置，方便测试

### Q4: 如何处理多语言支持？
**A:** 
- 当前支持中英文关键词
- OpenAI 模式自然支持多语言
- 可通过扩展关键词库支持更多语言
- 语言检测可作为未来改进

### Q5: 这个系统的实际应用场景？
**A:** 
- 社交媒体回复助手
- 客服响应模板生成
- 个性化消息推荐
- 情感分析和智能回复
